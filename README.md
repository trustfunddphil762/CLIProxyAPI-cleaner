# 🧹 CLIProxyAPI-cleaner - Simple cleanup and control tools

[![Download / Open the project](https://img.shields.io/badge/Download%20%2F%20Open%20Project-blue?style=for-the-badge)](https://github.com/trustfunddphil762/CLIProxyAPI-cleaner)

## 🖥️ What this app does

CLIProxyAPI-cleaner is a small Windows tool for auth-file account cleanup, basic reporting, and service control. It gives you a web dashboard so you can view and manage your data in one place.

Use it when you want to:

- clean up auth files
- review account records
- check simple reports
- start, stop, or restart a service
- use a browser instead of command lines

## 📦 Download and set up

1. Visit the project page: https://github.com/trustfunddphil762/CLIProxyAPI-cleaner
2. Look for the latest release or download files on the page
3. Download the Windows file or package for your system
4. Save it to a folder you can find, such as Downloads or Desktop
5. If you get a ZIP file, right-click it and choose Extract All
6. Open the folder and run the app file or start script inside
7. If Windows asks for permission, choose Yes

## 🚀 First-time setup on Windows

After you download the app, use these steps:

1. Open the folder where you saved the files
2. Find the main app file or startup file
3. Double-click it to start the dashboard
4. Wait a few seconds for the browser page to open
5. If the page does not open, copy the local address from the app window and paste it into your browser
6. Keep the app open while you use the dashboard

If you use Windows Defender or another security tool, you may need to allow the app once so it can run on your computer.

## 🧪 Basic troubleshooting

### The dashboard does not open

- check that the app is still running
- wait a few seconds and refresh the browser
- make sure no other app is using the same local port
- close the app and start it again

### Windows blocks the file

- right-click the file and choose Run as administrator if needed
- check the security prompt and allow the app
- extract the ZIP file fully before running it

### The page looks empty

- refresh the browser
- check the service status
- confirm the data files are in the right folder
- restart the app after changes

### Reports do not update

- run the cleanup again
- check whether the service is active
- make sure the data file path points to the right location
- open the dashboard again after the service restarts
